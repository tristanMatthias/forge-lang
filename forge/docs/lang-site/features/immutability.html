Immutability stable

immutable by default — let (frozen), mut (mutable), const (compile-time)

Variable bindings with let (immutable), mut (mutable), and const (compile-time constant)

Forge is immutable by default. Variables declared with `let` cannot be reassigned after initialization. This is not a convention or a lint rule; it is enforced by the compiler. Attempting to assign to a `let` binding produces error F0013 with a clear message explaining the immutability constraint.

Mutable bindings require the explicit `mut` keyword: `mut counter = 0`. This makes every mutation point visible in the code. When reading a function, you can immediately see which values might change by scanning for `mut` declarations. This is especially valuable in larger codebases where understanding data flow is critical.

Constants declared with `const` are even stricter: their values must be known at compile time, and they are inlined at every usage site. Use `const` for configuration values, mathematical constants, and fixed strings.

The immutable-by-default philosophy extends beyond variables to data structures. Structs created with `let` have immutable fields. The `with` expression creates modified copies rather than mutating in place. This approach eliminates shared mutable state, the root cause of countless bugs in imperative programs.

Syntax

let x = 1
mut y = 2
const Z = 3

Examples

const_binding.fg

/// # Const Binding
///
/// Compile-time constant with `const`.
///

fn main() {
  const MAX = 100
  println(MAX)
}

const_declaration.fg

/// # Const Declarations
///
/// Const values used in expressions.
///

fn main() {
  const PI_APPROX = 314
  const MULTIPLIER = 2
  println(string(PI_APPROX))
  println(string(PI_APPROX * MULTIPLIER + PI_APPROX * 4))
}

const_reassign.fg

/// # Const Cannot Be Reassigned
///
/// Attempting to reassign a const binding produces an error.
///

fn main() {
  const X = 42
  X = 99
}

destructure.fg

/// # Let Destructuring
///
/// Destructure tuples and structs with let.
///

fn main() {
  let (name, age) = ("hello", 42)
  println(name)
  println(age)
}

fn_param_immutable.fg

/// # Function Parameter Reassignment
///
/// Function parameters are immutable; reassigning them is an error.
///

fn double(x: int) -> int {
  x = x * 2
  return x
}

fn main() {
  println(string(double(5)))
}

global_mut.fg

/// # Global Mutable Variable
///
/// Top-level mut variable modified across function calls.
///

mut counter = 0

fn increment() {
  counter = counter + 1
}

fn main() {
  println(string(counter))
  increment()
  println(string(counter))
  increment()
  println(string(counter))
}

immutable_field_access.fg

/// # Immutable Field Access
///
/// Accessing fields of an immutable struct binding.
///

type Person = { name: string, age: int }

fn main() {
  let p = Person { name: "alice", age: 30 }
  println(p.name)
  println(string(p.age))
}

immutable_field_mutation.fg

/// # Immutable Field Mutation
///
/// Cannot assign to a field of an immutable struct variable.
///

type Point = { x: int, y: int }

fn main() {
  let p = Point { x: 1, y: 2 }
  p.x = 10
}

/// # Immutable Function Parameters

}

mut_in_loop.fg

/// # Mutable Variable in Loop
///
/// A mut variable accumulated across loop iterations.
///

fn main() {
  mut sum = 0
  mut i = 0
  while i < 100 {
    sum = sum + 1
    i = i + 1
  }
  println(string(sum))
}

mut_in_nested_scope.fg

/// # Mutable Variable Modified in Nested Scope
///
/// A mut variable can be modified from within a nested block.
///

fn main() {
  mut x = 10
  println(string(x))
  {
    x = 20
    println(string(x))
  }
  println(string(x))
}

mut_reassign.fg

/// # Mut Variable Reassign
///
/// Mutable variables can be reassigned multiple times.
///

fn main() {
  mut x = 1
  println(string(x))
  x = 2
  println(string(x))
  x = 3
  println(string(x))
}

reassignment_types.fg

/// # Variable Reassignment
///
/// Mutable variables can be reassigned, let bindings cannot.
///

fn main() {
  mut counter = 0
  counter = counter + 1
  counter = counter + 1
  println(string(counter))

  const MAX = 100
  println(string(MAX))
}

/// # Shadow Let With Mut
///
/// A let binding can be shadowed by a mut binding in the same scope.

  let x = 10
  println(string(x))

  println(string(x))
}

/// # Shadow Mut With Let Then Reassign
///

}
