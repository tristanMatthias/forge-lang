Component Events stable

event/on — hookable lifecycle events in components

Event declarations and user hooks in component templates (before_create, after_delete, etc.)

Component events declare hookable extension points in templates. A template can declare `event before_create(record)` to let users run custom logic before a record is created. Users hook into events with `on before_create(data) { validate(data) }` inside the component block.

Events follow a declaration-and-hook pattern. The template declares what events exist and what arguments they carry. User code attaches handlers to events it cares about. Events without handlers get no-op stubs, so unhandled events have zero runtime cost.

This provides a clean alternative to the middleware stacks and callback chains found in frameworks like Express or Django. Each event has a typed signature, so the compiler verifies that hook handlers accept the correct argument types.

The event system enables components to be customizable without inheritance or complex plugin architectures. A model component might offer `before_create`, `after_create`, `before_delete` events, letting users add validation, logging, or side effects without modifying the component template.

Syntax

event before_create(record)
on before_create(data) { }

Examples

hook_before_create_param.fg

/// # Before Create Hook Parameter
///
/// The `data` parameter in `on before_create(data)` should be bound
/// to the parsed record, allowing field access like `data.title`.
///

use @std.model.{model}

model Post {
  id: int @primary @auto_increment
  title: string

  on before_create(data) {
    println(data.title + " [modified]")
  }
}

fn main() {
  Post.create({ title: "Hello World" })
}

model_events.fg

/// # Component Events
///
/// Model components emit events like before_create and after_create.
/// Unhandled events get no-op stubs automatically.
///

use @std.model.{model}

model Item {
  id: int @primary @auto_increment
  name: string
}

fn main() {
  let id = Item.create({ name: "Widget" })
  let item = Item.get(id)
  println(item?.name ?? "not found")
  println(string(Item.count()))
}
