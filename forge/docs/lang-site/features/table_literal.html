Table Literal stable

table { cols | rows } — tabular data literals

Inline table literals that desugar to List<struct>

Table literals provide a concise syntax for defining tabular data using pipe-delimited columns. Instead of a list of structs or a list of lists, you write the data as a visual table directly in your source code, with `|` separating columns and each row on its own line.

This syntax is especially useful for test data, configuration tables, and any scenario where data is naturally two-dimensional. The visual alignment of columns makes the data easy to read and verify at a glance, unlike nested data structures that obscure the tabular nature of the data.

Table literals produce a typed list of records. The first row defines the column names and types, and subsequent rows provide the values. The compiler verifies that every row has the correct number of columns and that values match the declared types.

This feature is unique to Forge. While languages like Haskell have QuasiQuoters and Ruby has heredocs for embedding structured data, Forge's table literals are first-class syntax with full type checking and compile-time validation.

Syntax

table { col1 | col2; val1 | val2 }

Examples

basic.fg

/// # Basic table literal
///
/// Table literals desugar to List<struct>.
///

fn main() {
    let users = table {
        name    | age | active
        "alice" | 30  | true
        "bob"   | 25  | false
    }

    println(string(users.length))
    println(users[0].name)
    println(string(users[1].age))
    println(string(users[1].active))
}

composition_with_loops.fg

/// # Table Processing
///
/// Using table literals with loops for
/// structured data processing.
///

fn main() {
  let people = table {
    name      | age
    "Alice"   | 30
    "Bob"     | 25
    "Charlie" | 35
  }

  for row in people {
    println(`${row.name}: ${row.age}`)
  }
}

expressions.fg

/// # Table with expressions in cells
///
/// Cell values can be any expression, not just literals.
///

fn discount(price: float) -> float { price * 0.8 }

fn main() {
    let base = 100.0

    let products = table {
        name     | price      | discounted
        "Widget" | base       | discount(base)
        "Gadget" | base * 2.0 | discount(base * 2.0)
    }

    println(string(products[0].price))
    println(string(products[0].discounted))
    println(string(products[1].discounted))
}

table_access_last.fg

/// # Table Access Last Element
///
/// Access the last row by computed index.
///

fn main() {
  let users = table {
    name    | age
    "alice" | 30
    "bob"   | 25
    "carol" | 35
  }

  let last_idx = users.length - 1
  println(users[last_idx].name)
  println(string(users[last_idx].age))
}

table_computed_cells.fg

/// # Table with Computed Cell Values
///
/// Table cells can be arbitrary expressions including string templates.
///

fn main() {
  mut items: List<{label: string, value: int}> = []
  for i in 1..3 {
    items.push({ label: `item-${i}`, value: i })
  }
  println(string(items.length))
  println(items[0].label)
  println(items[1].label)
}

table_in_function.fg

/// # Table in Function
///
/// Table literal created and returned from a function.
///

fn make_users() -> List<{name: string, age: int}> {
    table {
        name    | age
        "alice" | 30
        "bob"   | 25
    }
}

fn main() {
    let users = make_users()
    println(string(users.length))
    println(users[0].name)
    println(users[1].name)
}
