let/mut/const — immutable-by-default bindings with type inference

Variable declarations with let, mut, const, and destructuring patterns

Variables in Forge are declared with `let` for immutable bindings, `mut` for mutable ones, and `const` for compile-time constants. Immutability is the default, which means `let x = 10` creates a binding that can never be reassigned. This design choice catches an entire class of bugs at compile time and makes code easier to reason about, since you always know a `let` binding holds the value it was initialized with.

Type inference means you rarely need to annotate types. The compiler figures out that `let name = "Alice"` is a `string` and `let count = 42` is an `int`. When you do want to be explicit, annotations go after the name: `let ratio: float = 3.14`. Mutable variables use `mut`: `mut counter = 0` followed by `counter = counter + 1`.

Constants declared with `const` must have values known at compile time. Unlike `let` bindings, constants are inlined everywhere they are used, so they carry zero runtime cost. Use constants for magic numbers, configuration values, and anything that should never change across the lifetime of the program.

If you are coming from JavaScript or Python, the key difference is that Forge variables are immutable by default. If you are coming from Rust, the model is similar but without lifetime annotations. If you are coming from Go, think of `let` as a stricter `:=` that forbids reassignment.

Syntax

let x = expr
mut x = expr
const X = expr

Examples

block_expression_value.fg

/// # Variable From Block Expression
///
/// A variable bound to the result of a block expression.
///

fn main() {
  let result = {
    let a = 10
    let b = 20
    a + b
  }
  println(string(result))
}

block_scope_isolation.fg

/// # Block Scope Isolation
///
/// Variables defined in a block are not accessible outside it.
///

fn main() {
  {
    let inner = 42
  }
  println(string(inner))
}

deep_shadowing.fg

/// # Deeply Nested Shadowing
///
/// Variables shadowed through multiple nesting levels restore correctly.
///

fn main() {
  let x = 1
  println(string(x))
  {
    let x = 2
    println(string(x))
    {
      let x = 3
      println(string(x))
    }
    println(string(x))
  }
  println(string(x))
}

if_else_expression_value.fg

/// # Variable From If-Else Expression
///
/// A variable bound to the result of an if-else expression.
///

fn main() {
  let x = 100
  let label = if x > 50 { "big" } else { "small" }
  println(label)
}

if_scope_isolation.fg

/// # If-Else Scope Isolation
///
/// Variables defined in if/else branches do not leak to the outer scope.
///

fn main() {
  if true {
    let x = 10
  } else {
    let x = 20
  }
  println(string(x))
}

/// # Let Without Initializer
///

  println(string(x))
}

list_destructure_rest.fg

/// # List Destructure With Rest
///
/// Destructure first elements from a list using rest syntax.
///

fn main() {
  let [first, second, ...rest] = [1, 2, 3, 4, 5]
  println(string(first))
  println(string(second))
  println(string(rest.length))
  for item in rest {
    println(string(item))
  }
}

///

/// # Variable Shadowing in Nested Scopes
///
/// Variables can be shadowed in nested blocks, and the outer

///

  println(string(x))
  {
    let x = 2
    println(string(x))
  }
  println(string(x))
}

///
/// Same variable name rebound to different types via shadowing.

fn main() {
  let x = 42

  println(string(x))

similar_names.fg

/// # Similar Variable Names
///
/// Variables with similar names should not interfere with each other.
///

fn main() {
  let x = 1
  let x1 = 2
  let x_ = 3
  let _x = 4
  println(string(x))
  println(string(x1))
  println(string(x_))
  println(string(_x))
}
