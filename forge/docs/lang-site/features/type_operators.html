without/only/partial — type-level field transformations

Type-level operators: without, with, only, as partial for deriving types from existing types

Type operators are compile-time transformations that derive new types from existing ones. Forge provides `without`, `only`, `partial`, and `with` for manipulating struct types. For example, `type CreateUser = User without { id }` creates a type with all User fields except `id`.

The `only` operator selects a subset of fields: `type UserName = User only { name, email }`. The `partial` operator makes all fields optional: `type UserUpdate = User partial`. The `with` operator adds or overrides fields: `type AdminUser = User with { role: string }`.

These operators are essential for API design, where you often need variations of a base type for different operations (create, update, response). Instead of manually maintaining parallel type definitions that drift out of sync, type operators derive the variations and keep them consistent automatically.

Type operators compose: `type CreateUser = User without { id } with { password: string }` removes the `id` field and adds a `password` field in a single declaration. This is similar to TypeScript's `Omit`, `Pick`, and `Partial` utility types but with cleaner syntax.

Type without field
Type only [fields]
partial Type

Examples

as_partial.fg

/// Test: as partial — all fields optional

type User = { name: string, email: string, age: int }
type UserUpdate = User only {name, email, age} as partial

fn main() {
  let update: UserUpdate = { name: "bob" }
  println(update.name ?? "none")
  println(update.email ?? "none")
}

chaining.fg

/// Test: chaining type operators

type User = { id: int, name: string, email: string, password: string, role: string }

type UpdateUser = User without {id, password} as partial

fn takes_update(u: UpdateUser) {
  println(u.email ?? "none")
}

fn main() {
  takes_update({ email: "new@test.com" })
}

composition_with_structs.fg

/// # Type Transformations
///
/// Using only to pick specific fields
/// for type-safe struct projections.
///

type User = { name: string, age: int, email: string }
type Summary = User only {name, age}

fn print_summary(u: Summary) {
  println(u.name)
  println(u.age)
}

fn main() {
  let user: Summary = { name: "alice", age: 30 }
  print_summary(user)
}

intersection_validate.fg

/// Test: intersection merges annotations from both types

type HasName = { name: string @min(1) @max(100) }
type HasEmail = { email: string @validate(email) }
type ContactInfo = HasName & HasEmail

fn main() {
  let result = validate({ name: "", email: "bad" }, ContactInfo)
  match result {
    Err(e) -> println(string(e.fields.length))
    Ok(_) -> println("should not reach")
  }
}

type_partial_usage.fg

/// # Partial Type in Function
///
/// Partial type used as a function parameter with some fields omitted.
///

type Config = { host: string, port: int, debug: bool }
type PartialConfig = Config only {host, port, debug} as partial

fn show_host(c: PartialConfig) {
  println(c.host ?? "none")
}

fn main() {
  show_host({ host: "localhost" })
  show_host({ port: 8080 })
}
