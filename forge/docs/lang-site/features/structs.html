Structs stable

struct declarations, literals, and field access

Type declarations (structs and aliases) and struct literal expressions

Structs define structural types with named fields. They are the primary way to group related data in Forge. Declaration syntax is: `type Point { x: int, y: int }`. Instances are created with literal syntax: `let p = Point { x: 10, y: 20 }`. Fields are accessed with dot notation: `p.x`.

Forge structs are immutable by default. To create a modified copy, use the `with` expression: `let q = p with { x: 30 }`. This creates a new struct with the specified fields changed and all others copied from the original. This approach encourages immutable data flow and makes it clear exactly which fields differ between two values.

Structs support shorthand field initialization when the variable name matches the field name: `let x = 10; let y = 20; Point { x, y }`. This reduces boilerplate when constructing structs from local variables with matching names.

Unlike classes in object-oriented languages, Forge structs carry no methods or inheritance. Behavior is attached through trait implementations, keeping data and behavior cleanly separated. This design scales better for large codebases and avoids the deep inheritance hierarchies that plague OOP systems.

Syntax

struct Name { field: type }
let x = Name { field: value }

Examples

deeply_nested_struct.fg

/// 5 levels of struct nesting (inline)
///

type L1 = { val: int }
type L2 = { inner: L1 }
type L3 = { inner: L2 }
type L4 = { inner: L3 }
type L5 = { inner: L4 }

fn main() {
  let x = L5 { inner: L4 { inner: L3 { inner: L2 { inner: L1 { val: 99 } } } } }
  println(string(x.inner.inner.inner.inner.val))
}

method_chain.fg

/// Method chaining on structs: a.method1().method2().method3()
///

type Num = { val: int }

impl Num {
  fn double(self) -> Num { Num { val: self.val * 2 } }
  fn add_one(self) -> Num { Num { val: self.val + 1 } }
  fn get(self) -> int { self.val }
}

fn main() {
  let n = Num { val: 5 }
  let result = n.add_one().double().double()
  println(string(result.get()))
}

nested_structs.fg

/// Nested structs: struct containing struct containing struct
///

type Inner = { value: int }
type Middle = { inner: Inner }
type Outer = { middle: Middle }

fn main() {
  let o = Outer { middle: Middle { inner: Inner { value: 42 } } }
  println(`deep value = ${o.middle.inner.value}`)
}

struct_as_fn_param.fg

/// Struct passed as function parameter and returned
///

type Point = { x: int, y: int }

fn translate(p: Point, dx: int, dy: int) -> Point {
  Point { x: p.x + dx, y: p.y + dy }
}

fn main() {
  let p = Point { x: 1, y: 2 }
  let p2 = translate(p, 2, 5)
  println(`(${p2.x}, ${p2.y})`)
}

struct_with_methods.fg

/// Struct with impl methods returning new struct
///

type Vec2 = { x: int, y: int }

impl Vec2 {
  fn add(self, other: Vec2) -> Vec2 {
    Vec2 { x: self.x + other.x, y: self.y + other.y }
  }
  fn sub(self, other: Vec2) -> Vec2 {
    Vec2 { x: self.x - other.x, y: self.y - other.y }
  }
  fn show(self) -> string {
    `(${self.x}, ${self.y})`
  }
}

fn main() {
  let a = Vec2 { x: 1, y: 2 }
  let b = Vec2 { x: 3, y: 4 }
  println(a.add(b).show())
  println(a.sub(b).show())
}
