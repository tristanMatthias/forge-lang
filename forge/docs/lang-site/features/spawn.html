Spawn stable

spawn { } — lightweight concurrent task execution

Concurrent execution with spawn { ... } blocks

The `spawn` keyword launches a concurrent task that executes independently from the spawning code. `spawn { expensive_computation() }` starts the computation and immediately continues with the next line. Tasks run concurrently, and their results can be communicated back through channels.

Spawn is the primary concurrency primitive in Forge. Rather than managing threads directly, you spawn lightweight tasks and communicate between them using channels. This follows the CSP (Communicating Sequential Processes) model, where shared state is replaced by message passing.

Spawned tasks share no mutable state with the spawning code. Any data needed by the task is captured at spawn time. This eliminates data races by construction, since there is no shared mutable memory to race on.

The model is similar to Go's goroutines and Erlang's processes. Tasks are lightweight enough to spawn thousands without performance concerns. Combined with channels and select, spawn provides a complete concurrent programming toolkit.

Syntax

spawn { body }

Examples

basic_spawn.fg

/// # Basic Spawn
///
/// Spawn a concurrent task that sends a value through a channel.
///

use @std.channel

fn main() {
  let ch = channel(1)

  spawn {
    ch <- "hello from spawn"
  }

  sleep(50)
  let msg = <- ch
  println(msg)
}

composition_with_channels.fg

/// # Concurrent Computation with Spawn
///
/// Spawning a background task and collecting
/// the result via a channel.
///

use @std.channel

fn main() {
  let ch = channel(1)

  spawn {
    let answer = 6 * 7
    ch <- string(answer)
  }

  sleep(50)
  let result = <- ch
  println(`result: ${result}`)
}

spawn_capture_multiple.fg

/// # Spawn Captures Multiple Variables
///
/// A spawned task captures several outer variables.
///

use @std.channel

fn main() {
  let name = "alice"
  let age = 30
  let ch = channel(1)

  spawn {
    ch <- `${name} is ${age}`
  }

  sleep(50)
  let msg = <- ch
  println(msg)
}

spawn_compute.fg

/// # Spawn with Computation
///
/// Spawned task does computation before sending result.
///

use @std.channel

fn main() {
  let ch = channel(1)

  spawn {
    mut sum = 0
    for i in 1..11 {
      sum = sum + i
    }
    ch <- string(sum)
  }

  sleep(50)
  let result = <- ch
  println(result)
}

spawn_multiple.fg

/// # Spawn Multiple Tasks
///
/// Spawn several tasks that each send a message through separate channels.
/// Each channel guarantees ordering.
///

use @std.channel

fn main() {
  let ch1 = channel(1)
  let ch2 = channel(1)
  let ch3 = channel(1)

  spawn { ch1 <- "first" }
  spawn { ch2 <- "second" }
  spawn { ch3 <- "third" }

  sleep(100)

  println(<- ch1)
  println(<- ch2)
  println(<- ch3)
}

spawn_sequential_channels.fg

/// # Spawn with Sequential Channel Reads
///
/// Multiple spawned tasks send to the same channel. Read all results.
///

use @std.channel

fn main() {
  let ch = channel(3)

  spawn { ch <- "a" }
  spawn { ch <- "b" }
  spawn { ch <- "c" }

  sleep(100)

  let m1 = <- ch
  let m2 = <- ch
  let m3 = <- ch
  mut count = 0
  if m1 != "" { count = count + 1 }
  if m2 != "" { count = count + 1 }
  if m3 != "" { count = count + 1 }
  println(string(count))
}

/// # Spawn with Shared State

}
