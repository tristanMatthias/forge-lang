F0001 — Unexpected token

The parser encountered a token it did not expect at this position.

Help: Check for missing punctuation, misspelled keywords, or extra characters.

This error occurs when the Forge parser encounters a token that doesn't fit the expected grammar at the current position. Common causes: - Missing semicolons or closing delimiters - Misspelled keywords (e.g., `fucntion` instead of `fn`) - Extra characters or symbols in the wrong place Example: ```forge fn main() { println("hello" // <- missing closing parenthesis } ```

F0002 — Unterminated string literal

A string literal was opened but never closed.

Help: Add a closing quote to terminate the string.

String literals must be closed with a matching quote character. Example of the error: ```forge let name = "hello ``` Fix: ```forge let name = "hello" ```

F0003 — Unterminated template literal

A template literal was opened but never closed.

Help: Add a closing backtick to terminate the template literal.

Template literals (backtick strings) must be closed with a matching backtick. Example of the error: ```forge let msg = `hello ${name} ``` Fix: ```forge let msg = `hello ${name}` ```

F0006 — Invalid numeric literal

A numeric literal could not be parsed as a valid number.

Help: Check that the number is well-formed (e.g., no double dots, valid digits).

This error occurs when the lexer encounters a sequence that looks like a number but cannot be parsed as a valid integer or float. Examples: ```forge let x = 12.34.56 // two decimal points let y = 99999999999999999999 // overflow ```

F0012 — Type mismatch

The type of an expression does not match what was expected.

Help: Check the types in your expression. You may need a conversion function like `string()` or `int()`.

This error occurs when the compiler expects one type but finds another. Example: ```forge let port: string = 8080 // int assigned to string ``` Fix: ```forge let port: string = string(8080) ```

F0013 — Cannot assign to immutable variable

Attempted to assign a new value to an immutable variable.

Help: Use `mut` instead of `let` if you need to reassign this variable.

Variables declared with `let` are immutable by default. To make a variable reassignable, use `mut` instead. Example of the error: ```forge let x = 42 x = 100 // error: x is immutable ``` Fix: ```forge mut x = 42 x = 100 // ok: x is mutable ```

F0014 — Wrong number of arguments

A function was called with the wrong number of arguments.

Help: Check the function signature and provide the correct number of arguments.

This error occurs when a function is called with more or fewer arguments than it expects. Example: ```forge fn add(a: int, b: int) -> int { a + b } fn main() { let result = add(1) // error: expects 2 arguments, got 1 } ``` Fix: ```forge fn main() { let result = add(1, 2) // correct: 2 arguments } ```

F0020 — Undefined variable

A variable was referenced that has not been defined in the current scope.

Help: Check for typos in the variable name, or define it before use.

This error occurs when you use a variable name that hasn't been declared in any accessible scope. Example: ```forge fn main() { println(string(x)) // error: x is not defined } ``` Fix: ```forge fn main() { let x = 42 println(string(x)) } ```

F0072 — Unknown annotation

An annotation was used that is not recognized as a core or provider annotation.

Help: Check for typos in the annotation name. Available core annotations: @min, @max, @validate, @pattern, @transform, @default.

This error occurs when you use an annotation that is not recognized by the compiler. Example of the error: ```forge type User = { name: string @required, } ``` Fix — use a valid annotation: ```forge type User = { name: string @validate(len > 0), } ```

F0073 — Annotation target mismatch

A type-level annotation was used on a field, or vice versa.

Help: Move the annotation to the correct position — type-level annotations go before fields, not on them.

This error occurs when an annotation is used on the wrong target. For example, @table is a type-level annotation that belongs at the top of a model block, not on individual fields. Example of the error: ```forge model User { name: string @table("custom") // error: @table is type-level } ``` Fix: ```forge model User { @table("custom") name: string } ```

F0074 — Provider annotation outside its context

A provider-scoped annotation was used outside its provider context.

Help: This annotation is only valid inside a model block. Use `model` instead of `type` for database features.

This error occurs when a model/provider annotation like @primary, @unique, @auto_increment, @hidden, or @owner is used on a plain type declaration instead of inside a model block. Example of the error: ```forge type UserInput = { name: string @primary, // error: @primary needs a model } ``` Fix: ```forge use @std.model.{model} model UserInput { name: string @primary } ```

F0801 — Unused variable

A variable was declared but never used.

Help: Prefix the variable name with `_` to suppress this warning, or remove the variable.

This warning is emitted when a variable is declared but never referenced. Unused variables may indicate dead code or a logic error. To suppress this warning, prefix the variable name with an underscore: ```forge let _unused = compute_something() ```

F9999 — Internal compiler error

The compiler encountered an unexpected internal error.

Help: This is a bug in the Forge compiler. Please report it at https://github.com/forge-lang/forge/issues

This error indicates a bug in the Forge compiler itself. The compiler encountered an unexpected state that it could not recover from. Please report this error with: 1. The source file that triggered it 2. The full error message 3. Your Forge compiler version (`forge version`)
